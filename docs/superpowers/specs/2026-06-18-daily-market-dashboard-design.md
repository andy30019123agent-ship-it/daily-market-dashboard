# 每日台美股戰略儀表板 — 設計規格

- 日期：2026-06-18
- 作者：Andy 的 AI 分身（與 Andy 共同腦力激盪）
- 狀態：待 Andy 審核

## 1. 目標

一個獨立的網頁「戰略儀表板」，每天台北時間 **18:43** 自動更新當日台股、美股的重點戰略資訊，並透過 Telegram 推送「重點摘要 + 連結」給 Andy。網頁保存每一天的內容，可用日期選單回看任何一天的歷史。

定位：幫 Andy 每天盤後快速掌握市場全局、輔助投資決策的「戰略總覽」。與既有的 `tw-stock-screener`（選股工具）**互相獨立、不整合、互不影響**。

## 2. 使用者與使用情境

- 單一使用者：Andy（非工程師）。
- 每天 18:43 收到 Telegram 訊息（摘要 + 連結）→ 手機瞄一眼掌握大方向 → 想看細節點連結進儀表板 → 偶爾回看過去某天的紀錄做對照。

## 3. 運作流程（每日自動）

排程於台北 18:43 觸發後，依序執行：

1. **抓硬數據**：台股、美股大盤指數與漲跌幅、成交量、恐懼貪婪指數、市場熱門個股（漲幅/爆量/資金流入）。
2. **蒐集軟情報**：上網蒐集當日川普動態及其他可能影響股市的消息、資金板塊流向解讀、熱門股緣由；由 AI 分身判讀整理成重點，**每條附來源連結**。
3. **處理財經日程**：本週若有重大日程 → 預告並分析可能影響；昨日已過的日程 → 整理事後結果與相關資料。
4. **生成當日資料檔**並併入歷史保存。
5. **建置並部署網頁**（更新今日內容、保留歷史）。
6. **Telegram 推送**：3~5 句今日最重點摘要 + 儀表板連結。
7. **失敗處理**：任一步驟出錯時，仍 Telegram 通知 Andy「今天產報告失敗 + 原因」，不靜默失敗。

「誰來做判讀」：採 **甲方案** — 每日排程喚醒 Andy 的 AI 分身（Claude cloud 排程），由分身親自蒐集、判讀、生成報告。不需額外 API 金鑰；代價是每天用掉一些 Andy 的 Claude 訂閱用量。

## 4. 網頁內容（6 模組 + 日期回看）

主頁 = 今日儀表板，一打開即當日最新。頂部有日期選單／小月曆，可往回翻看任何一天；每天內容永久保存。

1. **今日總覽**：台股加權、美股道瓊／標普／那斯達克／費半（SOX）的收盤與漲跌幅、成交量；恐懼貪婪指數（情緒一眼看完）。
2. **板塊資金流向 + 熱門個股**：資金流入／流出的板塊解讀；市場熱門個股（當日漲幅榜／爆量／資金流入）。
3. **川普及其他影響股市消息**：當日重點消息，**每條附來源連結**。
4. **本週重大日程提醒 + 影響分析**：本週若有重大日程（聯準會 FOMC、四巫日、CPI／非農、重要財報等）先預告並分析可能影響。
5. **昨日已過日程的事後回顧**：前一日剛過的重大日程，整理結果與相關資料。
6. **今日綜合研判**：利多 / 利空 / 隱憂，做為收尾。

## 5. 資料來源策略

分兩類（最終以實作時實測驗證為準）：

**硬數據（客觀、可直接抓）**
- 台股加權指數、成交量、漲跌家數、熱門個股：TWSE 證交所 OpenAPI（免費、無額度限制）。
- 美股指數（道瓊／標普／那斯達克／SOX）：免費資料源（如 stooq 免費 CSV 或同級來源），實作時驗證穩定度。
- 美股恐懼貪婪指數：CNN Fear & Greed（取其公開資料端點）。
- 台股情緒：無官方對應指數，實作時以替代指標（漲跌家數、融資餘額等）呈現或明確標註，待確認。

**軟情報（需蒐集 + 判讀）**
- 川普動態及其他影響股市消息、資金板塊解讀、熱門股緣由：AI 分身執行時上網蒐集（網路搜尋），判讀整理並附來源連結。

**財經日程**
- 已知行事曆（FOMC、四巫日、CPI／非農、台股除權息與重要財報季等）：部分可預先建表，搭配網路搜尋補當週實際日期。

## 6. 歷史保存與反查

- 每天一份資料檔（如 `data/YYYY-MM-DD.json`），永久保存。
- 一份索引列出所有可查日期，供前端日期選單使用。
- 前端依選定日期載入對應資料檔渲染；預設載入最新一天。

## 7. 技術選型（初步，實作前可再議）

- **前端**：靜態網頁（傾向 Vite + React，與既有專案技術棧一致，利於維護與後續美化）。資料以 JSON 檔提供，純前端渲染。
- **排程**：Claude cloud 排程（routine / cron），每日台北 18:43 觸發 AI 分身執行完整流程。
- **資料抓取／生成**：Python 或 Node 腳本負責硬數據；軟情報由 AI 分身於執行時產生。
- **部署**：GitHub Pages（新建獨立 repo，與 tw-stock-screener 分開）。
- **通知**：Telegram（沿用現有 channel）。

## 8. 設計風格

依 Andy 偏好，UI 一律同時套用 `frontend-design` 與 `ui-ux-pro-max` 兩套技能。儀表板需資訊密度高但清晰、手機可讀（盤後常用手機看），情緒指標與漲跌用顏色快速辨識。

## 9. 成本

- 基礎建設（抓數據、網頁、部署、Telegram）：0 元。
- 軟情報判讀：不需現金，但每天耗用 Andy 的 Claude 訂閱用量一趟。

## 10. 風險與待驗證項目

- **免費資料源穩定度**：美股指數、F&G、台股情緒指標的免費來源可能變動或被擋，需實作時驗證並備援。
- **cloud 排程穩定度**：每日自主執行需可靠；務必有失敗 Telegram 通知。
- **台股恐懼貪婪指數**：無官方來源，呈現方式待定（替代指標或標註）。
- **軟情報品質**：本質為 AI 判讀，附來源連結供 Andy 查證；偶有時間差或角度偏差屬預期。

## 11. 明確排除（YAGNI）

- 不與 tw-stock-screener 整合（Andy 決定）。
- 不做盤中即時／推播（僅每日盤後一次）。
- 不做多使用者、登入、帳號系統。
- 不做手動「立即重抓」按鈕（避免金鑰曝光，與選股專案同原則）。
