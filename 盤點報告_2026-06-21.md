## 1. BUG（實際與潛在、邊界、資料正確性）

### 🔴 BUG-1：`SystemExit` 不被 `except Exception` 捕獲，失敗通知靜默

**位置：** `scripts/auto_daily.py` - `main()` 第 202 行

`main()` 只捕獲 `Exception`，但以下情況都用 `raise SystemExit(...)` 拋出，會直接逃過 try/except，導致 **不發 Telegram 失敗通知、GitHub Actions 只顯示紅燈但你毫不知情**：

- `pick_partial()`：找不到 partial.json（line 78）
- `send_tg()`：缺少 `TG_BOT_TOKEN`（line 126）
- `send_tg()`：Telegram API 回傳失敗（line 136）
- `validate_day()`：schema 驗證未過（line 165）
- `gen_soft_openai.py`：缺少 `OPENAI_API_KEY`（line 116）

**怎麼修：** 把 `main()` 的 `except Exception as e:` 改為 `except (Exception, SystemExit) as e:`，或把所有 `raise SystemExit(...)` 改為 `raise RuntimeError(...)`。

---

### 🟡 BUG-2：排程每天執行（含週六日），空燒 Alpha Vantage 配額

**位置：** `.github/workflows/daily.yml` cron `'36 10 * * *'`

週六日台股休市，但排程仍執行並呼叫 18 次 AV（SOXX + 6 個股 + 11 類股 ETF）。notify_state dedup 雖然阻止重複推播，但 **18 次 AV quota 已白白燒掉**，當天若再手動補跑（18 次）就會超額（18+18=36 > 25）。

STATUS.md 說「一～五」，但 cron 並無週一到週五限制（缺少 `1-5` 第 5 個欄位）。

**怎麼修：** cron 改為 `'36 10 * * 1-5'`（只有週一到週五執行）。週六日若有美股補假日資料需求，可另外用 `workflow_dispatch` 手動觸發。

**怎麼修（短期）：** 先修 cron 為週一到週五（BUG-2），立刻省去週六日 2 天 × 18 次。**長期：** 把 11 類股 ETF 改為每週一次（或快取一天不重抓），每日改為 1+6=7 次，留足手動補跑空間。

## 7. 相依套件風險

### 🟡 套件-1：vitest 包含 1 個 critical + 1 個 high 漏洞（開發環境）

```
CRITICAL: vitest ≤2.1.9（依賴 @vitest/mocker, vite, vite-node）
HIGH: vite ≤6.4.2（透過 vitest）
MODERATE: esbuild ≤0.24.2（透過 vite）
```

漏洞核心：esbuild dev server 可被任意網站請求（SSRF-like，`GHSA-67mh-4wv8-2f99`）。

**影響評估：** vitest 是 devDependency，CI 雖 `npm ci` 安裝，但只跑 `npm run build`，不啟動 vitest dev server。**生產環境不受影響**，本機若跑 `npm test` 時有風險。

**怎麼修：** `npm audit fix --force`（會升級 vitest 到 4.x，為 breaking change，需確認測試通過）。或短期接受風險（僅影響本機開發）。

## 8. 建置與上線

### 🟡 建置-1：workflow 無失敗通知 step，只靠 Python 腳本自行推 Telegram

GitHub Actions 本身沒有 `on-failure` 通知機制，完全依賴 auto_daily.py 的 `except Exception` 捕獲後推 Telegram。但如 BUG-1 所示，SystemExit 不被捕獲，且 fetch_hard_data 步驟獨立（見風控-1）。

**怎麼修：** 在 workflow 加一個 `if: failure()` 的 step：
```yaml
- name: 失敗通知
  if: failure()
  env:
    TG_BOT_TOKEN: ${{ secrets.TG_BOT_TOKEN }}
  run: |
    python -c "
    import os, urllib.request, urllib.parse
    token = os.environ['TG_BOT_TOKEN']
    data = urllib.parse.urlencode({'chat_id':'-5127072553','text':'⚠️ 台美股排程失敗，請檢查 GitHub Actions'}).encode()
    urllib.request.urlopen(f'https://api.telegram.org/bot{token}/sendMessage', data=data, timeout=10)
    "
```
